## 1. 产品概述

### 1.1 产品定位

基于现有 LinkedIn 自动投递能力，扩展支持国内主流求职平台（Boss直聘、智联招聘），为国内求职者提供智能化、批量化的简历投递服务。

### 1.2 目标用户

**主要用户群体**：
- 20-35 岁技术类求职者（应届生、1-5 年工作经验）
- 需要批量投递简历的职场人士
- 希望提高投递效率的求职者

**用户特征**：
- 熟悉国内主流招聘平台操作
- 有一定技术背景，能够使用本地工具
- 重视投递效率和成功率
- 关注账号安全和合规性

### 1.3 核心价值

- **效率提升**：从手动投递 5-10 个/小时提升到 30-50 个/小时
- **智能匹配**：AI 自动分析岗位匹配度，优先投递高匹配职位
- **风险可控**：支持黑白名单、投递前审核、频率控制
- **数据追踪**：完整的投递历史记录和效果分析

## 2. 用户画像与场景分析

### 2.1 典型用户画像

**用户 A - 应届毕业生小李**
- 年龄：22 岁，计算机专业应届生
- 痛点：缺乏工作经验，需要大量投递增加面试机会
- 需求：批量投递、自动填写表单、追踪投递状态
- 使用场景：每天晚上投递 30-50 个职位，重点关注互联网大厂和创业公司

**用户 B - 跳槽者老王**
- 年龄：28 岁，3 年 Python 后端经验
- 痛点：在职找工作时间有限，需要高效投递
- 需求：精准匹配、黑名单过滤（避开现公司）、投递前审核
- 使用场景：每周投递 2-3 次，每次 20 个左右，重点关注薪资和公司规模

**用户 C - 转行者小张**
- 年龄：26 岁，从传统行业转行做 AI 应用开发
- 痛点：简历不够匹配，需要大量投递积累经验
- 需求：自动优化简历、智能回答附加问题、投递记录分析
- 使用场景：每天投递 40-60 个职位，关注培训机会和初创公司

### 2.2 核心使用场景

**场景 1：批量投递 Boss 直聘职位**
1. 用户在 Boss 直聘登录账号
2. 在系统中配置投递参数（关键词、城市、薪资范围）
3. 启动自动投递，系统自动搜索、筛选、投递
4. 投递时自动填写"打招呼"内容和自我介绍
5. 实时显示投递进度和成功率
6. 完成后生成投递报告

**场景 2：智联招聘简历投递**
1. 用户在智联招聘登录账号并上传简历
2. 配置投递策略（岗位类型、公司规模、行业）
3. 系统自动搜索匹配职位
4. 自动选择合适的简历版本投递
5. 处理附加问题（工作年限、期望薪资等）
6. 记录投递历史和企业反馈

**场景 3：多平台统一管理**
1. 用户同时配置 Boss 和智联两个平台
2. 系统根据岗位特点自动选择投递平台
3. 统一的黑白名单管理（避免重复投递）
4. 跨平台的投递数据统计和分析
5. 一键导出所有平台的投递记录

## 3. 功能需求

### 3.1 Boss 直聘自动投递（P0 - 必须有）

#### 3.1.1 登录与会话管理
- **登录方式**：
  - 支持手机验证码登录
  - 支持账号密码登录
  - 会话持久化（保存 Cookie，下次免登录）
  - 多账号管理（支持切换不同账号）

- **安全机制**：
  - 使用 undetected-chromedriver 避免被检测
  - 随机延迟模拟人类行为（2-5 秒）
  - 检测验证码并暂停等待人工处理
  - 检测账号异常状态（被限制、被封禁）

#### 3.1.2 职位搜索与筛选
- **搜索参数**：
  - 关键词（支持多个关键词组合）
  - 工作地点（支持多城市）
  - 薪资范围（如：15K-30K）
  - 工作经验要求（应届、1-3 年、3-5 年等）
  - 学历要求（本科、硕士等）
  - 公司规模（0-20 人、20-99 人等）
  - 融资阶段（未融资、天使轮、A 轮等）

- **智能筛选**：
  - 过滤已投递职位（避免重复）
  - 过滤黑名单公司
  - 优先投递白名单公司
  - 根据匹配度排序（AI 分析简历与 JD 匹配度）

#### 3.1.3 自动投递与沟通
- **投递流程**：
  1. 点击"立即沟通"按钮
  2. 自动填写打招呼内容
  3. 可选：附加自我介绍
  4. 提交投递
  5. 记录投递结果

- **打招呼内容生成**：
  - 模板库（预设 10+ 个模板）
  - AI 生成（根据岗位 JD 和简历生成个性化内容）
  - 用户自定义（支持变量替换：{职位名称}、{公司名称}等）
  - 长度控制（Boss 限制 100 字以内）

- **自我介绍**：
  - 从简历中提取关键信息
  - 突出与岗位相关的经验
  - 控制在 200 字以内

#### 3.1.4 投递控制
- **频率控制**：
  - 每次投递间隔 3-8 秒（随机）
  - 每小时最多 50 个职位
  - 每天最多 200 个职位
  - 检测到风控时自动暂停

- **投递前审核**：
  - 可选：每次投递前暂停，显示职位详情
  - 用户确认后继续投递
  - 支持跳过当前职位

- **异常处理**：
  - 检测"今日沟通已达上限"提示
  - 检测"该职位已下线"提示
  - 检测验证码并暂停
  - 自动重试失败的投递（最多 3 次）

### 3.2 智联招聘自动投递（P1 - 重要）

#### 3.2.1 登录与会话管理
- **登录方式**：
  - 账号密码登录
  - 手机验证码登录
  - 会话持久化
  - 多账号管理

- **简历管理**：
  - 检测已上传的简历列表
  - 支持选择不同简历版本投递
  - 检测简历完整度（提示补充信息）

#### 3.2.2 职位搜索与筛选
- **搜索参数**：
  - 关键词
  - 工作地点
  - 薪资范围
  - 工作经验
  - 学历要求
  - 公司性质（国企、外企、民营等）
  - 公司规模
  - 行业类型

- **智能筛选**：
  - 过滤已投递职位
  - 过滤黑名单公司
  - 匹配度评分（AI 分析）
  - 优先级排序

#### 3.2.3 自动投递
- **投递流程**：
  1. 点击"申请职位"按钮
  2. 选择合适的简历版本
  3. 填写附加信息（如有）
  4. 提交投递
  5. 记录投递结果

- **附加信息处理**：
  - 期望薪资（从配置中读取）
  - 到岗时间（立即、1 个月内等）
  - 工作年限（从简历中提取）
  - 自我评价（从简历中提取或 AI 生成）

#### 3.2.4 附件上传
- **简历附件**：
  - 支持 PDF、Word 格式
  - 自动检测是否需要上传附件
  - 从本地选择简历文件上传
  - 记录上传成功/失败状态

### 3.3 通用功能（P0 - 必须有）

#### 3.3.1 多平台统一配置
- **配置界面**：
  - 平台选择（Boss、智联、或两者）
  - 投递数量限制
  - 搜索关键词
  - 工作地点
  - 薪资范围
  - 黑白名单

- **配置保存**：
  - 保存为配置文件（JSON）
  - 支持多套配置方案
  - 快速切换配置

#### 3.3.2 投递历史追踪
- **记录内容**：
  - 投递时间
  - 平台名称
  - 职位名称
  - 公司名称
  - 投递状态（成功/失败/跳过）
  - 失败原因
  - 职位链接

- **数据存储**：
  - SQLite 本地数据库
  - 支持导出为 CSV/Excel
  - 支持按时间、平台、状态筛选

#### 3.3.3 数据统计分析
- **实时统计**：
  - 当前任务进度（已投递/总数）
  - 成功率
  - 失败原因分布
  - 平均投递速度

- **历史统计**：
  - 每日投递数量趋势
  - 各平台投递分布
  - 成功率趋势
  - 高频失败原因

- **可视化展示**：
  - 投递进度条
  - 成功率饼图
  - 投递趋势折线图
  - 平台分布柱状图

#### 3.3.4 黑名单/白名单管理
- **黑名单**：
  - 公司黑名单（不投递指定公司）
  - 职位黑名单（不投递包含特定关键词的职位）
  - 支持模糊匹配
  - 支持批量导入

- **白名单**：
  - 优先投递白名单公司
  - 白名单职位优先级更高
  - 支持设置优先级权重

## 4. 非功能需求

### 4.1 性能要求

- **投递速度**：
  - Boss 直聘：30-50 个/小时
  - 智联招聘：20-40 个/小时
  - 响应时间：每次操作 < 10 秒

- **并发能力**：
  - 支持单账号单平台投递
  - 暂不支持多账号并发（避免资源竞争）

- **资源占用**：
  - 内存占用 < 500MB
  - CPU 占用 < 30%
  - 磁盘占用 < 100MB（不含浏览器缓存）

### 4.2 安全要求

- **反爬虫机制**：
  - 使用 undetected-chromedriver
  - 随机 User-Agent
  - 随机延迟（2-8 秒）
  - 模拟鼠标移动和滚动

- **账号安全**：
  - 不存储明文密码
  - Cookie 加密存储
  - 检测异常登录提示
  - 自动暂停避免封号

- **数据安全**：
  - 本地存储，不上传云端
  - 敏感信息脱敏（日志中不显示密码）
  - 支持数据清除功能

### 4.3 可用性要求

- **成功率**：
  - 正常情况下投递成功率 > 90%
  - 遇到验证码时自动暂停，不计入失败
  - 遇到风控时自动停止，保护账号

- **稳定性**：
  - 连续运行 2 小时不崩溃
  - 异常自动恢复（网络中断、页面加载失败等）
  - 支持断点续投（任务中断后可继续）

- **易用性**：
  - 配置界面简洁直观
  - 实时显示投递进度
  - 错误提示清晰易懂
  - 提供详细的使用文档

### 4.4 兼容性要求

- **操作系统**：
  - Windows 10/11（主要支持）
  - macOS 12+（次要支持）
  - Linux（暂不支持）

- **浏览器**：
  - Chrome 100+（推荐）
  - Edge 100+（兼容）

- **Python 版本**：
  - Python 3.9+

## 5. 技术约束

### 5.1 国内网络环境

- **网络稳定性**：
  - 需要稳定的网络连接
  - 支持网络中断后自动重连
  - 超时时间设置：30 秒

- **CDN 加速**：
  - 国内招聘网站使用 CDN，加载速度较快
  - 需要处理动态加载的内容（Ajax）

### 5.2 平台反爬虫机制

#### Boss 直聘
- **检测机制**：
  - User-Agent 检测
  - 行为特征检测（点击速度、鼠标轨迹）
  - IP 频率限制
  - 设备指纹识别

- **应对策略**：
  - 使用 undetected-chromedriver
  - 随机延迟和随机行为
  - 保持登录状态（避免频繁登录）
  - 控制投递频率

#### 智联招聘
- **检测机制**：
  - 滑块验证码
  - 行为验证（点击、滚动）
  - 会话有效期检测

- **应对策略**：
  - 人工处理滑块验证码（暂停等待）
  - 模拟真实用户行为
  - 定期刷新会话

### 5.3 验证码处理

- **验证码类型**：
  - 图片验证码（少见）
  - 滑块验证码（常见）
  - 短信验证码（登录时）

- **处理方案**：
  - 检测到验证码时自动暂停
  - 弹窗提示用户手动处理
  - 处理完成后自动继续
  - 记录验证码出现频率

### 5.4 IP 限制

- **限制情况**：
  - 短时间内大量请求可能被限制
  - 异地登录可能触发风控

- **应对策略**：
  - 控制请求频率
  - 使用本地 IP（不使用代理）
  - 保持登录状态稳定

## 8. 风险与应对

### 8.1 技术风险

#### 风险 1：平台反爬虫升级
- **影响**：投递功能失效
- **概率**：中
- **应对**：
  - 持续监控平台变化
  - 快速迭代反检测策略
  - 提供降级方案（半自动投递）
  - 建立用户反馈机制

#### 风险 2：验证码频繁出现
- **影响**：投递效率降低
- **概率**：中
- **应对**：
  - 优化行为模拟，降低触发概率
  - 提供验证码自动识别（OCR）
  - 人工处理验证码时提供友好提示

#### 风险 3：浏览器驱动兼容性问题
- **影响**：无法启动浏览器
- **概率**：低
- **应对**：
  - 锁定 Chrome 和驱动版本
  - 提供自动更新驱动功能
  - 提供详细的环境配置文档

### 8.2 合规风险

#### 风险 1：违反平台服务条款
- **影响**：账号被封、法律风险
- **概率**：中
- **应对**：
  - 在文档中明确说明使用风险
  - 提供免责声明
  - 建议用户控制投递频率
  - 不提供商业化服务（仅供学习）

#### 风险 2：用户数据泄露
- **影响**：隐私泄露、法律责任
- **概率**：低
- **应对**：
  - 本地存储，不上传云端
  - 敏感信息加密
  - 提供数据清除功能
  - 开源代码，接受社区审查

### 8.3 运营风险

#### 风险 1：用户过度使用导致封号
- **影响**：用户投诉、项目声誉受损
- **概率**：中
- **应对**：
  - 在文档中明确建议投递频率
  - 默认限制投递数量
  - 提供风险提示
  - 建立用户社区，分享最佳实践

#### 风险 2：维护成本高
- **影响**：功能失效、用户流失
- **概率**：中
- **应对**：
  - 模块化设计，便于维护
  - 建立自动化测试
  - 鼓励社区贡献
  - 提供详细的开发文档

#### 风险 3：竞品出现
- **影响**：用户流失
- **概率**：高
- **应对**：
  - 持续优化用户体验
  - 增加差异化功能（AI 分析、匹配度评分）
  - 建立用户社区
  - 保持开源优势
