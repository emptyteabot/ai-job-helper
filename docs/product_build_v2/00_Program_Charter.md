# 00 Program Charter

更新时间：2026-02-17
负责人：Product Lead（多角色协作）

## 1. 项目目标

在中国市场打造“上传简历 -> 获得可投递岗位 -> 优化简历 -> 面试准备 -> 反馈闭环”的 AI 求职产品，形成可增长、可复用、可融资叙事。

## 2. 核心成功指标（North Star + 业务）

- North Star：每周“有效投递会话数”
- 业务指标：
  - 分析成功率 >= 95%
  - 真实岗位返回率 >= 85%（核心关键词样本）
  - 岗位点击率 >= 25%
  - 搜索->投递转化率 >= 8%
  - 7 日留存 >= 18%

## 3. 当前状态评估

- 产品状态：Beta（可演示，不可大规模推广）
- 关键短板：
  - 真实岗位源稳定性不足
  - 部分地区访问稳定性不足
  - 增长实验与用户反馈闭环未形成周节奏

## 4. 阶段划分

- Phase A（1 周）：稳定产品底座
  - 完成真实岗位源兜底策略和质量闸门
- Phase B（1-2 周）：提升转化路径
  - 首页与工作台统一转化导向体验
- Phase C（2 周）：增长验证
  - 跑 3 轮增长实验，输出留存与转化提升报告
- Phase D（持续）：市场化扩张
  - 渠道拓展、案例沉淀、商业化试点

## 5. 组织与角色

- 需求分析师：目标用户、需求优先级、验收定义
- 架构设计师：系统边界、服务治理、技术路线
- UI/UX 设计师：体验规范、信息架构、转化路径
- 前端工程师：页面交付、埋点接入、性能优化
- 后端平台工程师：接口契约、稳定性与容错
- 数据/岗位源工程师：岗位数据质量与实时性
- QA 工程师：测试策略、回归与发布质量
- DevOps/SRE：部署、监控、故障处理
- 安全合规：隐私、安全、数据边界
- 验收与发布经理：发布准入、回滚机制
- Marketing/GTM：增长策略与市场节奏

## 6. 决策机制

- 每日站会：15 分钟
- 每周评审：目标、数据、风险、是否继续投放
- 决策优先级：真实可用性 > 稳定性 > 美观 > 新功能

## 7. 风险与预案

- 风险：岗位源风控导致结果为空
  - 预案：企业 API + 云缓存双通道
- 风险：域名访问抖动
  - 预案：Cloudflare 前置 + 多地域入口
- 风险：AI 输出跑偏
  - 预案：质量闸门 + 模板兜底 + 监控告警

## 8. 发布门槛

满足 `10_验收与发布_文档.md` 中全部 P0/P1 条件后，才进入市场化放量。
