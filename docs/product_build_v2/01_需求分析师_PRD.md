# 01 需求分析师 PRD

更新时间：2026-02-17
Owner：需求分析师

## 1. 问题定义

中国求职用户在投递阶段主要痛点：
- 简历内容与岗位要求不匹配
- 岗位信息噪声大、筛选成本高
- 面试准备碎片化，缺少结构化辅导

## 2. 目标用户（ICP）

- B2C：20-35 岁技术类求职者（应届/1-5 年经验）
- B2B（后续）：高校就业中心、训练营、求职服务机构

## 3. 用户旅程

1. 进入首页，理解产品价值
2. 上传简历或粘贴文本
3. 获取岗位匹配与可投递链接
4. 获取优化简历和面试问题
5. 点击岗位、投递、回填反馈

## 4. 核心需求（P0/P1/P2）

P0（必须）
- 上传简历并稳定解析
- 返回真实可点击岗位（或明确空结果）
- 返回优化简历（必显示）
- 返回面试准备建议
- 全链路埋点可观测

P1（应有）
- 多城市/多关键词策略
- 岗位质量评分和去重
- 结果导出与分享
- 增长看板实时展示

P2（可选）
- 登录与历史记录
- 个性化投递策略
- 企业版团队管理

## 5. 非功能需求

- 可用性：主流程成功率 >= 95%
- 性能：首屏 < 2.5s，核心接口 P95 < 3s（不含外部源超时）
- 可靠性：错误有可读提示，不返回假数据
- 可观测性：关键事件可追踪

## 6. 指标体系

- 激活：上传转分析比
- 价值：岗位点击率、简历下载率
- 结果：搜索->投递转化率
- 留存：7 日留存、复访分析率

## 7. 验收标准（需求侧）

- 用户可以在 5 分钟内完成一次完整求职分析
- 结果输出可理解、可执行、可保存
- 无真实岗位时必须透明告知，不可伪造

## 8. 待确认清单

- 首批目标城市（建议：北上深杭广）
- 首批岗位族（建议：Python/后端/AI应用）
- 商业化模式优先级（订阅 vs 服务包）
