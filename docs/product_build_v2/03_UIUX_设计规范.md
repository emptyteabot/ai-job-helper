# 03 UI UX 设计规范

更新时间：2026-02-17
Owner：UI/UX 设计师

## 1. 体验目标

- 第一屏 3 秒内理解价值
- 1 次上传完成核心分析
- 结果层级清晰、可立即行动

## 2. 视觉原则

- 风格：白底黑字、极简、留白充足
- 字号：标题大、正文可读、移动端不拥挤
- 文案：以“用户收益”表达，避免工程口吻

## 3. 信息架构

- 首页：价值主张 -> 进入工作台 -> 反馈收集
- 工作台：输入区 -> 增长看板 -> 岗位 -> 优化简历 -> 分析详情
- 投资看板：仅面向内部/投资人

## 4. 文案规范

- 禁用：技术实现过程、内部术语堆砌
- 必须：用户动作 + 结果收益 + 下一步建议
- 空状态：明确原因 + 可执行下一步

## 5. 交互规范

- 上传、分析、下载均要有状态反馈
- 岗位链接点击后记录事件，不打断用户
- 错误信息必须中文可读

## 6. 可用性检查清单

- 移动端首屏不溢出
- 核心按钮可见且可点击
- 文本区支持复制与导出
- 色彩对比满足可读性

## 7. 后续设计任务

- A/B 两版首页 Hero（转化导向）
- 工作台卡片化结果页优化
- 岗位卡片增加“匹配理由”简版
