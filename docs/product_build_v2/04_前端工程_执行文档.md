# 04 前端工程 执行文档

更新时间：2026-02-17
Owner：前端工程师

## 1. 目标

实现稳定、可转化的工作台体验，确保结构化数据渲染与增长埋点完整。

## 2. 页面范围

- `static/index.html`：首页
- `static/app_clean.html`：工作台
- `static/investor.html`：投资看板

## 3. 核心接口契约（必须）

- `POST /api/process`
  - 使用 `recommended_jobs` 渲染岗位卡片
  - 兼容 `job_recommendations` 文本展示
- `GET /api/business/public-proof`
  - 渲染增长 KPI
- `POST /api/business/event`
  - 发送用户事件

## 4. 前端事件埋点

- `workspace_opened`
- `resume_process_started`
- `resume_process_success`
- `resume_process_failed`
- `job_link_click`
- `result_download`

## 5. 质量要求

- 不渲染入口页/假岗位
- API 异常时不崩溃，有明确提示
- 所有交互按钮有 loading 或状态

## 6. 待办

- 岗位卡片增加“发布时间/来源”二级信息
- 增加岗位排序（匹配度/新鲜度）
- 新增移动端断点优化（<= 390px）

## 7. 验收点

- 首次加载可见价值文案
- 上传并分析后，岗位区可渲染结构化数据
- 下载、点击埋点可在 KPI 中反映
