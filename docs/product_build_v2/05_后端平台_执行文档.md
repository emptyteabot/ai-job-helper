# 05 后端平台 执行文档

更新时间：2026-02-17
Owner：后端平台工程师

## 1. 目标

保证 `process` 主流程稳定、输出结构化、可观测、可审计。

## 2. 已落地能力

- `process_response.v2` 输出结构
- 质量门禁（低质量文案拦截）
- 严格真实岗位过滤（默认禁用入口回退）
- 增长事件写入与公开 KPI 聚合

## 3. 核心策略

- 无真实岗位时返回空结果 + 明确提示
- 禁止默认返回演示岗位或入口链接
- 外部源失败不导致服务崩溃

## 4. 后端待办

- provider 并发搜索 + 质量评分合并
- `/api/process` 结果缓存（同简历短时复用）
- 错误码标准化（provider timeout / anti-bot / parse-failed）

## 5. 监控指标

- `resume_processed.ok`
- `quality_gate.passed`
- `real_jobs_count`
- `api_error` 分布

## 6. 配置约束

- 默认：`ALLOW_CN_PORTAL_FALLBACK=false`
- 可选：配置企业 API 提升稳定岗位返回率

## 7. 验收标准

- `/api/process` 输出通过契约校验
- 错误不抛 HTML，不抛非 JSON
- 关键事件写入成功率 >= 99%
