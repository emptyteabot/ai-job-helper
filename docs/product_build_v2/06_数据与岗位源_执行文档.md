# 06 数据与岗位源 执行文档

更新时间：2026-02-17
Owner：数据/岗位源工程师

## 1. 目标

把“真实岗位返回率”做到可预测、可维护、可运营。

## 2. 数据源分层

- L1：企业级招聘 API（首选，稳定）
- L2：云端岗位缓存（定时推送）
- L3：无浏览器搜索（DDG/Bing HTML）
- L4：入口页链接（仅应急开关，默认禁用）

## 3. 数据质量规则

- 必须包含 `title + link`
- 去重规则：`link > id > title|company`
- 过滤入口页、演示页、seed 页
- 优先保留 Boss/猎聘/智联/51job/拉勾域名

## 4. 新鲜度与覆盖

- 新鲜度目标：48 小时内岗位占比 >= 70%
- 覆盖目标：核心 5 城 + 3 岗位族

## 5. 运营指标

- real_jobs_count
- no_real_jobs_rate
- provider_success_rate
- duplicate_rate

## 6. 本周动作

- 接入一个稳定企业 API（优先）
- 建立日更缓存任务
- 输出 provider 健康报告
