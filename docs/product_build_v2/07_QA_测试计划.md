# 07 QA 测试计划

更新时间：2026-02-17
Owner：QA 工程师

## 1. 测试目标

确保“上传 -> 分析 -> 岗位 -> 下载 -> 埋点”主链路稳定可回归。

## 2. 测试范围

- API 契约测试
- 岗位过滤逻辑测试
- 输出质量门禁测试
- 前端关键交互测试
- 部署后冒烟测试

## 3. 用例优先级

P0
- `/api/process` 返回 JSON 且包含 v2 字段
- 无真实岗位时不出现假岗位
- 优化简历模块始终显示

P1
- 埋点事件可写入
- KPI 面板可读取
- 下载文件可用

P2
- 多语言兼容
- 非核心浏览器适配

## 4. 自动化

- 单测：`pytest -q`
- 发布前：健康检查 + 核心接口回归

## 5. 缺陷分级

- P0：主流程不可用、错误数据、崩溃
- P1：核心体验受损、可绕过
- P2：体验优化项

## 6. 发布闸门

- P0=0，P1<=2 且有明确修复计划
- 自动化测试全部通过
- 线上冒烟用例通过
