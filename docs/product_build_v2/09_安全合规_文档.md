# 09 安全合规 文档

更新时间：2026-02-17
Owner：安全与合规负责人

## 1. 数据分级

- 高敏：用户简历原文、联系方式
- 中敏：行为事件、反馈文本
- 低敏：匿名聚合指标

## 2. 基本原则

- 最小化采集：只收必要字段
- 最小化存储：明确保留周期
- 默认脱敏：日志不写入敏感全文

## 3. 必须措施

- 环境变量管理所有密钥
- API 输入长度与格式校验
- 后台管理接口鉴权
- 错误输出不暴露内部栈细节

## 4. 合规边界

- 明示用途：用于求职分析和产品改进
- 用户可请求删除数据（后续实现）
- 对外展示仅用匿名聚合统计

## 5. 安全测试

- 基础注入与异常输入测试
- 依赖漏洞扫描（每周）
- 发布前安全检查清单
