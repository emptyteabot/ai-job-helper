# 12 多Agent协作机制

更新时间：2026-02-17
Owner：项目经理

## 1. 协作原则

- 统一目标：先解决真实可用，再追求规模增长
- 并行执行：模块内并行，接口先行
- 快速闭环：每日同步、每周复盘

## 2. 协作节奏

- 每日 10:00：站会（15 分钟）
- 每日 18:00：进度同步（阻塞与风险）
- 每周五：版本评审 + 数据复盘

## 3. 交付接口

- 需求 -> 设计：用户故事 + 验收标准
- 设计 -> 前端：页面规范 + 组件说明
- 前后端：API 契约文档 + Mock
- 开发 -> QA：测试清单 + 回归路径
- QA -> 发布：缺陷清单 + 风险建议
- 发布 -> 增长：指标看板 + 监控链接

## 4. 变更管理

- P0 变更：当日评审、当日落地
- P1 变更：纳入周迭代
- P2 变更：进入需求池

## 5. 决策规则

- 数据冲突时以线上指标为准
- 方案冲突时以“用户结果”优先
- 任何角色可提阻塞，PM 必须闭环
