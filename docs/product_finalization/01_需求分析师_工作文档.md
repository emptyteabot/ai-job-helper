# 需求分析师工作文档

## 角色目标

把“想法”转化为可执行、可验收、可排期的需求清单。

## 核心产出

- PRD v1.0
- 用户画像与 JTBD
- 用户旅程图（首次访问、上传简历、查看岗位、下载结果）
- 需求优先级矩阵（P0/P1/P2）
- 验收标准文档（Given-When-Then）

## 需求优先级

P0:

- 用户可上传简历并成功分析
- 推荐岗位为真实可点击链接
- 系统在无实时抓取时提供明确兜底路径

P1:

- 分析结果可下载
- 页面移动端适配
- 结果区可折叠、可快速定位

P2:

- 个性化推荐理由解释
- 历史分析记录

## 需求评审标准

- 是否可测试
- 是否可追踪指标
- 是否有明确边界
- 是否有失败场景定义

## 协作接口

- 向架构设计师提交需求边界
- 向 UI/UX 提交用户旅程和信息优先级
- 向 QA 提交验收用例基线
