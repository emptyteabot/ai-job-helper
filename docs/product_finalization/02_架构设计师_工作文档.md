# 架构设计师工作文档

## 角色目标

提供可扩展、可运维、可降级的整体技术架构。

## 架构原则

- 优先保证主路径可用
- 依赖外部平台时必须有降级方案
- 服务接口稳定优先于局部最优实现

## 当前架构分层

- Presentation: `static/index.html`, `static/app_clean.html`
- API: `web_app.py`
- Domain: `app/core/*`
- Integration: `app/services/*`, `app/services/job_providers/*`

## 关键技术决策

- 中国市场岗位优先策略（Boss/猎聘/智联/51job/拉勾）
- 多级降级链路（实时抓取 -> 搜索回退 -> 站点入口回退）
- 推荐岗位输出必须包含可行动链接

## 质量属性

- 可用性：>= 99%
- 可观测性：关键接口有日志、指标、错误上下文
- 可恢复性：支持快速回滚与配置开关

## 风险清单

- 外部搜索风控导致结果不稳定
- 第三方依赖变更导致解析失效
- 大文本处理造成高延迟

## 协作接口

- 与后端平台工程师定义 API 契约
- 与 DevOps 定义部署、监控和告警指标
- 与安全合规确认数据处理边界
