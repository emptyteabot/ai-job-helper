# UI/UX 设计师工作文档

## 角色目标

把产品从“功能可用”升级为“体验可卖”。

## 设计方向

- 风格：极简、克制、白底黑字
- 信息密度：首屏只展示关键动作与关键结果
- 交互原则：一步一反馈，失败可恢复

## 设计规范

- 统一字体: IBM Plex Sans / Mono
- 统一留白: 8px 基线网格
- 统一按钮层级: 主按钮 1 个，次按钮 <= 3 个
- 统一状态提示: loading / success / error

## 页面策略

首页:

- 只保留价值主张 + 主 CTA
- 移除非必要卡片和说明块

工作台:

- 上部输入区
- 中部岗位卡片区
- 下部分析结果折叠区

## 体验验收

- 首屏 5 秒内看懂产品做什么
- 核心路径点击不超过 3 步
- 移动端无需横向滚动

## 协作接口

- 给前端交付组件规格和交互说明
- 给 QA 交付视觉与交互验收清单
