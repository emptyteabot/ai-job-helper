# 前端工程师工作文档

## 角色目标

稳定实现极简界面和核心交互，保证性能与可用性。

## 代码范围

- `static/index.html`
- `static/app_clean.html`

## 实施要求

- 保持现有 API 绑定不破坏
- 组件命名清晰，ID 保持后端兼容
- 关键交互有错误提示和空状态

## 性能目标

- 首屏资源体积可控
- 长文本渲染不阻塞核心按钮操作
- 用户操作反馈 <= 150ms

## 可用性要求

- 拖拽上传和文件上传一致
- 失败时可重试
- 下载结果格式稳定

## 工程规范

- 避免重复样式
- 不引入无必要前端依赖
- 所有文案与状态可快速替换

## 协作接口

- 与后端确认响应字段兼容
- 与 UI/UX 对齐样式和交互细节
- 与 QA 对齐冒烟测试脚本
