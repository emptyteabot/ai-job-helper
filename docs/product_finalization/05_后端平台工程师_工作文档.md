# 后端平台工程师工作文档

## 角色目标

保障 API 服务稳定、可观测、可扩展。

## 代码范围

- `web_app.py`
- `app/core/*`

## 关键职责

- 维护核心接口稳定性（`/api/process`, `/api/jobs/search`）
- 统一错误码和错误文案
- 优化长任务处理和超时策略

## 接口治理

- 固定响应结构：`success`, `data/error`, `provider_mode`, `warning`
- 非 JSON 响应归零
- 关键接口保留向后兼容字段

## 稳定性目标

- 核心接口可用性 >= 99%
- 5xx 比例 < 1%
- 接口超时比例 < 3%

## 运维配合

- 输出健康检查增强项
- 输出请求链路日志字段
- 支持灰度与配置开关

## 协作接口

- 向 QA 提供接口契约和 mock 示例
- 向 DevOps 提供 SLI/SLO 指标定义
