# QA 测试工程师工作文档

## 角色目标

在上线前发现阻断问题，降低线上回归风险。

## 测试范围

- 功能测试：上传、分析、推荐、下载
- 接口测试：核心 API 响应结构与异常分支
- 兼容测试：Chrome/Edge、移动端
- 回归测试：版本迭代后关键路径

## 关键用例

- 无简历输入时的错误处理
- 推荐岗位为空时的提示和回退行为
- 推荐岗位链接可点击且来源正确
- 长文本结果渲染与下载完整性

## 质量闸门

- P0 缺陷必须为 0
- P1 缺陷可控且有 workaround
- 核心路径自动化冒烟通过

## 测试交付

- 测试报告（通过率、阻断项、残余风险）
- 缺陷单列表（优先级 + 所属团队）
- 发布建议（Go / Conditional Go / No-Go）

## 协作接口

- 与产品确认验收标准
- 与开发确认修复 SLA
- 与发布经理对齐上线窗口
