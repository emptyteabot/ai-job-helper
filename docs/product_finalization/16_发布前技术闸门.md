# 发布前技术闸门

## 目标

在 Go/No-Go 前做统一技术检查，避免“发布成功但功能不可用”。

## 必跑项

1. 语法检查

```bash
python -m py_compile web_app.py
python -m py_compile app/services/real_job_service.py
```

2. 自动化测试

```bash
pytest -q
```

3. Smoke 检查

```bash
python scripts/smoke_check.py
```

如果检查线上环境:

```bash
SMOKE_BASE_URL=https://ai-job-hunter-production-2730.up.railway.app python scripts/smoke_check.py
```

## 通过标准

- 测试全部通过
- `/api/ping`, `/api/version`, `/api/ready`, `/api/health` 全部 200
- `/api/ready` 返回 `status=ready`

## 失败处理

- 任一失败项触发 No-Go
- 走回滚或修复后重试，不允许带风险强推
