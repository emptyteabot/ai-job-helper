# 实时岗位数据怎么接

这个项目支持两种“实时岗位”思路：

1. **走第三方招聘聚合 API（推荐）**
- 例：Jooble（需要申请 `JOOBLE_API_KEY`），后端会直接返回带 `link` 的岗位列表，你点开链接后手动投递。

2. **走搜索引擎 API 返回岗位链接（备选）**
- 例：Bing Web Search（`BING_SEARCH_API_KEY`）。但 Bing Search APIs 已宣布在 2025-08-11 退休，新账号通常拿不到 Key。

关于 Boss/猎聘/智联/51job：
- 这些站点的“官方实时岗位搜索 + 自动投递”的开放 API，面向个人开发者通常不可直接申请，一般要走商务合作/企业权限/白名单。
- 因此产品上更现实的做法是：**实时搜到 URL -> 前端跳转 -> 你登录后手动点投递**（合规、稳定、不会卡验证码）。
